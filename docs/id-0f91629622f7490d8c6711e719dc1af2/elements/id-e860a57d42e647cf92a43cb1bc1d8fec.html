Implementation Factor Catalog ()
https://pubs.opengroup.org/togaf-standard/adm-techniques/chap06.html#tag_06_01 The technique of creating an Implementation Factor catalog can be used to document factors impacting the architecture Implementation and Migration Plan. The catalog should include a list of the factors to be considered, their descriptions, and the deductions that indicate the actions or constraints that have to be taken into consideration when formulating the plans. Factors typically include: Risks (ADM Risks https://pubs.opengroup.org/togaf-standard/adm-techniques/chap09.html) Issues (=Constraints) Assumptions Dependencies (=Constraints) Actions (=Course of Action) Impacts (=Objective)
